# 기기 설정 인벤토리 동기화 설계

## 목표

Token Deck 계정에 연결된 각 기기에서 Codex, Claude, Gemini가 사용하는 스킬, MCP, 플러그인 목록을 비교하고 안전하게 가져올 수 있게 한다.

## 개인정보 경계

클라우드에는 다음 메타데이터만 저장한다.

- 기기 식별자, 목록 스키마 버전, 수집·갱신 시각과 내용 비교용 해시.
- 도구 종류와 공급사.
- 스킬·MCP·플러그인 이름.
- 플러그인 버전과 마켓플레이스 식별자.
- 설치·활성 상태.
- MCP 전송 방식과 비밀 설정 필요 여부.
- 사용자·시스템·마켓플레이스 같은 수집 출처 분류.
- 자동 가져오기 가능 여부와 비밀 설정·로컬 경로·미지원 같은 제한 사유 분류.

다음 정보는 기기 밖으로 보내지 않는다.

- API 키, OAuth 토큰, 쿠키와 환경변수 값.
- MCP 명령·인수·URL 원문.
- 스킬 본문, 스크립트와 자산.
- 프로젝트 및 사용자 폴더의 전체 경로.
- 로컬 수집 경고의 원문.
- 자동 가져오기 제한 사유의 원문.

## 데이터 흐름

1. Rust 네이티브 수집기가 세 도구의 사용자 설정과 설치 목록을 읽는다.
2. 수집기는 허용된 필드만 포함한 `DeviceInventory`를 반환한다.
3. 사용자가 현재 기기의 목록 동기화를 켠 경우에만 프론트엔드가 동기화 전용 허용 목록으로 한 번 더 변환한다.
4. Supabase에는 사용자와 기기마다 최신 스냅샷 한 행만 저장한다.
5. 기기 화면은 원본 기기 스냅샷과 현재 기기 인벤토리를 비교한다.
6. 사용자가 항목을 선택하고 검토를 승인하면 Rust가 지원되는 항목만 현재 기기에 적용한다.

## 가져오기 범위

자동 가져오기는 안전하게 재현 가능한 범위로 제한한다.

- Codex와 Claude의 마켓플레이스 플러그인은 공식 CLI를 별도 인수로 실행해 설치 또는 활성화한다.
- 이미 설치된 Gemini 확장 기능은 활성화할 수 있다.
- MCP는 이름과 상태만 비교한다. 명령과 비밀값을 동기화하지 않으므로 다른 기기에 없는 MCP는 직접 설정해야 한다.
- 사용자 제작 스킬은 이름만 비교한다. 본문을 동기화하지 않으므로 다른 기기에 없는 스킬은 직접 설치해야 한다.

모든 적용은 현재 기기에만 수행한다. 다른 기기에 원격으로 설정을 밀어 넣지 않는다.

자동 갱신을 끄면 그 기기에서 새 스냅샷 업로드만 중단된다. 마지막 스냅샷은 같은 계정의 기기 비교를 위해 계정 데이터에 유지된다.

## 저장소 구조

`device_setting_snapshots`는 `(user_id, device_id)`를 기본 키로 사용한다. 항목별 행을 유지하지 않고 최신 스냅샷을 원자적으로 덮어써서 삭제된 로컬 항목도 다음 스냅샷에 자연스럽게 반영한다.

RLS는 로그인 사용자 자신의 행만 조회·삽입·수정할 수 있게 제한한다. 클라이언트 요청은 `user_id`를 보내지 않고 Supabase의 `auth.uid()` 기본값만 사용한다.

## 장애 격리

설정 스냅샷 테이블이 아직 배포되지 않았거나 로컬 도구 조사에 실패해도 기존 토큰 사용량 수집과 동기화는 계속된다. 설정 목록 오류는 별도 상태로 표시한다.
